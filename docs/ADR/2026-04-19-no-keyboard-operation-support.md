# ADR-014: キーボード操作対応を実施しない

## Status

**Proposed** (2026-04-19)

## Context

v6ロードマップにおいて、アクセシビリティ向上の一環として「キーボード操作完全対応」を検討していた。これは Command Palette的なインターフェースやショートカットキーを 통해、マウスなしで拡張機能を利用できるようにする提案だった。

## Decision

**キーボード操作対応（Command Palette、ショートカットキー）は実施しない。**

## Reason

1. **製品の本質との整合性**
   - 本拡張は「ユーザーが50%以上スクロールし、5秒以上読んだWebページを記録する」ことが本質
   - 記録の条件判定にブラウジング中のユーザーの行動（スクロール、滞在時間）を利用
   - ユーザーはブラウジング中にマウス/トラックパッドを既に使用しており、キーボード操作は必須でない

2. **実装コスト対効果**
   - キーボードショートカットの実装と維持コスト
   - 対象ユーザー層の小ささ
   - 本質機能への集中を優先

## Consequences

### Positive

- 開発リソースを本質機能に集中できる
- コードベースの複雑さを抑えられる
- テスト対象を減らせる

### Negative

- パワーユーザー（キーボードだけで操作したい層）への対応が抜ける
- 一部のユーザーにとって効率が落ちる可能性
