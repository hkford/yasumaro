## 実装詳細（記録）

### entrypoints 構成

```
entrypoints/
  background/index.ts         defineBackground + dynamic import
  content/index.ts            defineContentScript（旧 loader.ts）
  content-extractor.ts        defineUnlistedScript（旧 content/extractor.ts）
  popup/index.html + main.ts  5本の import を集約
  options/index.html + main.ts  i18n + dashboard import
  permissions/index.html + main.ts  privacy import
  offscreen.html              unlisted page
```

### WXT 出力構造

```
dist/chromium-mv3/
  manifest.json
  popup.html / options.html / permissions.html / offscreen.html
  background.js
  content-extractor.js
  content-scripts/content.js
  chunks/*.js / assets/*.css
  icons/ / _locales/ / data/ / PRIVACY.md
```

### 変更されたランタイムパス

| 旧 | 新 |
|----|-----|
| `content/extractor.js` | `content-extractor.js` |
| `dashboard/dashboard.html` | `options.html` |
| `privacy/privacy.html` | `permissions.html` |
| offscreen: `src/offscreen/offscreen.html` | `offscreen.html` |

### 判明した制約・ハマりポイント

- `entrypoints/content/loader.ts` → WXT に認識されない。`content/index.ts` に rename が必須
- `entrypoints/content/extractor.ts` → `content/` 内の非 `index` ファイルは無視される。`entrypoints/` 直下に `content-extractor.ts` として配置
- `optional_host_permissions` に `<all_urls>` + `host_permissions` にも `<all_urls>` → Chrome が redundant 警告
- Node v24 + htmlparser2 v10 → `EPERM: lstat` エラー。`overrides: { htmlparser2: "^12.0.0" }` で解消
