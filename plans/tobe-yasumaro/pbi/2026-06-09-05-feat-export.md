# PBI: データエクスポート機能（.db / Markdown / CSV）

## ユーザーストーリー
**yasumaroユーザーとして**、蓄積したブラウジング履歴を .dbファイル・Markdown・CSV形式でエクスポートしたい、なぜなら他のPCへの移行や他ツールへのデータ連携、Obsidianへの手動流し込みができるようにしたいから。

## ビジネス価値
- ポータビリティの担保により「データロックイン」への不安を解消し、ユーザー信頼を高める
- Obsidian連携のワークフローをサポートし、既存obsidian-weaveユーザーの移行を促進する

```gherkin
  When ユーザーが2件をチェックして「Markdownエクスポート」をクリックする
  Then 2件分のMarkdownファイル（または1ファイルに2エントリ）がダウンロードされる
  And Markdownにはタイトル・URL・要約・タグ・日時が含まれている
  And Obsidian互換のフロントマター（YAML形式）が付与されている

Scenario: 全ログをCSVエクスポートする
  Given SQLiteに500件のブラウジングログが存在する
  When ユーザーが「全件CSVエクスポート」をクリックする
  Then yasumaro_export_YYYYMMDD.csv がダウンロードされる
  And CSVにはurl, title, summary, tags, created_at, domain の列が含まれる
  And UTF-8 BOM付きでExcelで正しく開ける

Scenario: .dbファイルをインポートして既存データとマージする
  Given 別のPCからエクスポートした yasumaro.db がある
  When ユーザーが「インポート」から .db ファイルをアップロードする
  Then インポートファイルのデータが既存SQLiteにマージされる（URLの重複は無視）
  And 「N件インポートしました（M件は重複でスキップ）」というメッセージが表示される
```

## 受け入れ基準
- [ ] エクスポート形式: `.db`（OPFS → Blob → download）、Markdown、CSV の3種
- [ ] `.db` エクスポート: wa-sqliteの `sqlite3_serialize` APIでバイナリを取得してダウンロード
- [ ] `.db` インポート: ファイル選択 → wa-sqliteへロード → 既存DBへUPSERT
- [ ] Markdownフォーマット:
  ```markdown
  ---
  title: "記事タイトル"
  url: https://example.com
  date: 2026-06-09
  tags: [TypeScript, 入門]
  ---
  ## 要約
  AI要約の内容...
  ```
- [ ] CSVはUTF-8 BOM付き（Excel対応）
- [ ] スター付きレコードはCSV/Markdownエクスポート時に「保護対象」として一括削除から除外される
- [ ] エクスポートUIはダッシュボードの設定パネル内に配置

### E2Eテスト（手動確認）
- エクスポートした.dbファイルをDB Browser for SQLiteで開いてデータ確認

- CSVフォーマッター: カンマ・改行を含むフィールドのクォート処理

## 技術的考慮事項
- 依存関係: Phase 1（SQLite基盤）完了が前提
- `.db` バイナリダウンロード: `URL.createObjectURL(blob)` → `<a download>` で実現
- wa-sqliteの `sqlite3_serialize` がOPFS上のDBファイルをメモリにコピーして返す
- 大容量ファイル（数百MB）のダウンロードはServiceWorkerでstream処理が必要な場合あり（当面は100MB以内を想定）

## 実装者向け注記

### 現状コードの確認
```bash
grep -rn "export\|download\|blob" src/
grep -rn "Blob\|createObjectURL" src/
```

### 落とし穴
- OPFSのファイルを直接 `<a download>` でダウンロードはできない。必ず `sqlite3_serialize` でメモリに展開してからBlobに変換すること
- CSVのカンマ・ダブルクォート・改行を含むフィールドは RFC 4180 に従ってエスケープすること
- インポート時のURL重複チェックは `INSERT OR IGNORE` または `ON CONFLICT DO NOTHING` で対応
