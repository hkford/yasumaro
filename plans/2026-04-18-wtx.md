## 判明した WXT 出力構造

```
dist/chromium-mv3/
  manifest.json
  popup.html            ← entrypoints/popup/index.html から（popup/index.html ではない）
  options.html          ← entrypoints/options/index.html から
  permissions.html      ← entrypoints/permissions/index.html から
  offscreen.html        ← entrypoints/offscreen.html から（unlisted page）
  background.js
  content-scripts/
    content.js          ← entrypoints/content/index.ts から
  chunks/
    *.js
  assets/
    popup-*.css
    options-*.css
    permissions-*.css
  icons/
  _locales/
  data/
  PRIVACY.md
```

**manifest の実際の値:**
- `action.default_popup`: `"popup.html"`（`"popup/index.html"` ではない）
- `background.service_worker`: `"background.js"`
- `content_scripts[0].js`: `["content-scripts/content.js"]`

### Phase 8: ビルド確認・手動確認

**ステータス:** ユーザー作業待ち

```bash
npm run build
# dist/chromium-mv3/ に出力される
```

Chrome に `dist/chromium-mv3/` をロードして以下を確認:
- popup 表示・設定保存・動作
- dashboard (options) 表示・各パネル動作
- content script 動作（ページ訪問時の自動記録）
- offscreen document（Local AI 機能）
- privacy ページ表示

**offscreen URL 確認（重要）:**

`src/background/service-worker.ts` 内の `chrome.offscreen.createDocument` の `url` パラメータを確認:

```bash
grep -n "offscreen" src/background/service-worker.ts
```

WXT が生成するパスは `offscreen.html`（ルート）なので、
service-worker 側で `chrome.runtime.getURL('offscreen.html')` と参照しているか確認。

## リスクマップ（残り）

| リスク | 詳細 | 対策 |
|--------|------|------|
| offscreen URL 不一致 | service-worker の chrome.offscreen URL と WXT 生成パスが違う | WXT ビルド後に `dist/chromium-mv3/offscreen.html` の存在確認 + service-worker.ts の URL 参照を修正 |
| content script ESM 動作 | ESM format で Chrome に正しくインジェクトされるか不明 | Chrome で実証確認。失敗したら `defineContentScript` の設定を調整 |
| web_accessible_resources パス | WXT 出力では `content/loader.js` → `content-scripts/content.js` に変わった | wxt.config.ts の web_accessible_resources を WXT 出力パスに合わせて更新（Phase 9 前に確認） |
| dashboard の i18n 二重 import | `src/dashboard/dashboard.ts` が i18n を import 済みの場合、entrypoints 側との二重 import が問題になる可能性 | 動作確認で問題があれば `entrypoints/options/main.ts` から i18n import を削除 |
| entrypoints/ 内の孤立アイコン | `entrypoints/popup/icons/`, `entrypoints/options/icons/` が不要なコピー | Phase 10 で削除 |
